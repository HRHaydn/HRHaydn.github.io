Data Types

Representing Data Types


Why do we need data types?
A computer needs to know what data type it is storing because they are processed in different ways. A computer does not have the instinctive ability to recognise the different types of data, so we need to tell them what to do.

All data types are stored as binary. The main data types we use:

Type Description Example
Character Single letter, digit, symbol or control code. S, g, 7, &
String A string of alphanumeric characters. hat, Fg7tY6, %7&*j
Boolean One of two values. True or False.
Integer Whole number values with no decimal part. 6, -12, 9, 143
Real Numbers with decimal or fractional parts. 12.3, -18.63, 3.14

Representing Text
In order for text to be readable by all computer systems, an agreed method of representing characters and strings is important. This is the role of ASCII (American Standard Code for Information Interchange). Originally based on an 8-bit binary pattern using seven bits plus a single parity bit and was able to represent 128 separate characters. The extended version uses 8-bits and can represent 256 separate characters.

With the introduction of a need for high amounts of characters, Unicode was introcuced with 16-bits to allow more than 65,000 characters, however even this 16-bit restriction was later removed. The original ASCII table still takes up the first 256 numeric values of Unicode.


Boolean Data
Boolean is a data type that can only be one of two values: True or False, using 1 represent True and 0 to represent False. The data only requires one bit but is often stored in a byte for convenience. Boolean data types are often used to flag if an event has occurred.

Representing Integers in Binary


Positive Integers

When we write a number in base 10 (otherwise denary), we simply combine a quantity of 1's, 10's, 100's and so on to represent the value. In base 2 (otherwise known as binary), the collumns are based on 2 rather than 10.

Column Value 128 64 32 16 8 4 2 1
Binary Number 1 0 0 1 1 0 1 1

So the binary number 10011011 is the equivalent of 128+16+8+2+1 = 155 in denary.

Negative Integers
With negative integers, there are two approaches to representing a negative value. These are "Sign and Magnitude" and "Two's complement".

Sign and Magnitude:
In this approach, the most significant bit (MSB) is used to store the binary value of either 0 for positive or 1 for negative.
For example the binary value for negative 46 is shown below:

Column Value Sign Bit 64 32 16 8 4 2 1
1 0 1 0 1 1 1 0

This produces the binary value of 10101110 in an 8 bit arrangement.

Two's complement:
While the above method is fine, the processing required is quite complicated so there is a better approach. This is to make the MSB a negative value.
This produces these new headings:

Column Value -128 64 32 16 8 4 2 1

To produce the value of -103 we record -128 + 25 or:

Column Value -128 64 32 16 8 4 2 1
1 0 0 1 1 0 0 1

This produces the binary number 10011001.
