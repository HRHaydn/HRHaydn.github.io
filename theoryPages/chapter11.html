Data Types

Representing Data Types


Why do we need data types?
A computer needs to know what data type it is storing because they are processed in different ways. A computer does not have the instinctive ability to recognise the different types of data, so we need to tell them what to do.

All data types are stored as binary. The main data types we use:

Type Description Example
Character Single letter, digit, symbol or control code. S, g, 7, &
String A string of alphanumeric characters. hat, Fg7tY6, %7&*j
Boolean One of two values. True or False.
Integer Whole number values with no decimal part. 6, -12, 9, 143
Real Numbers with decimal or fractional parts. 12.3, -18.63, 3.14

Representing Text
In order for text to be readable by all computer systems, an agreed method of representing characters and strings is important. This is the role of ASCII (American Standard Code for Information Interchange). Originally based on an 8-bit binary pattern using seven bits plus a single parity bit and was able to represent 128 separate characters. The extended version uses 8-bits and can represent 256 separate characters.

With the introduction of a need for high amounts of characters, Unicode was introcuced with 16-bits to allow more than 65,000 characters, however even this 16-bit restriction was later removed. The original ASCII table still takes up the first 256 numeric values of Unicode.


Boolean Data
Boolean is a data type that can only be one of two values: True or False, using 1 represent True and 0 to represent False. The data only requires one bit but is often stored in a byte for convenience. Boolean data types are often used to flag if an event has occurred.

Representing Integers in Binary


Positive Integers

When we write a number in base 10 (otherwise denary), we simply combine a quantity of 1's, 10's, 100's and so on to represent the value. In base 2 (otherwise known as binary), the collumns are based on 2 rather than 10.

Column Value 128 64 32 16 8 4 2 1
Binary Number 1 0 0 1 1 0 1 1

So the binary number 10011011 is the equivalent of 128+16+8+2+1 = 155 in denary.

Negative Integers
With negative integers, there are two approaches to representing a negative value. These are "Sign and Magnitude" and "Two's complement".

Sign and Magnitude:
In this approach, the most significant bit (MSB) is used to store the binary value of either 0 for positive or 1 for negative.
For example the binary value for negative 46 is shown below:

Column Value Sign Bit 64 32 16 8 4 2 1
1 0 1 0 1 1 1 0

This produces the binary value of 10101110 in an 8 bit arrangement.

Two's complement:
While the above method is fine, the processing required is quite complicated so there is a better approach. This is to make the MSB a negative value.
This produces these new headings:

Column Value -128 64 32 16 8 4 2 1

To produce the value of -103 we record -128 + 25 or:

Column Value -128 64 32 16 8 4 2 1
1 0 0 1 1 0 0 1

This produces the binary number 10011001.

Special Relationship
When we represent large binary numbers such as 1000110011101111 we can easily convert them to hexadecimal. We start by first dividing them into nibbles (4-bits).

1000 1100 1110 1111


We then take each binary nibble and represent it with the corresponding hexadecimal value.

1000 1100 1110 1111
8 9 A B


By then taking these characters and simply slotting them together, we have correctly represented the binary number 1000110011101111 as 89AB in hex.

Images


Pretty Pixels
Everything in a computer and all its data must be represented in 1's and 0's and this is true even for images. Take for example a simplistic black and white image:


Each colour of an image is stored as a binary number. In the black-and-white image above, each pixel is either black or white.
You need a binary value for each different colour. As each pixel is either black or white, this image can be encoded with a value of 0 for white and 1 for black.
If we used two bits for each pixel then we could store 2^2 = 4 colours. For three bits 2^3 = 8 colours ,eight bits 2^8 = 256 colours and on and on.
The number of bits used for each dot/pixel is referred to as the colour depth.

A common misconception is that resolution is size of the image. For example a 500x500 image. However resolution is in fact the number of pixels or dots per unit. This is given the unit dpi (dots per inch) or ppi (pixels per inch).

With all this information about the resolution and colour depth it can be useful to store it so that we can look it up if needs be. The data about data is called metadata.

Bitmaps
Bitmap images are widely used on digital cameras, smartphones and online. Common bitmap image file types include JPEG, GIF and PNG. Bitmaps are also known as pixelmaps or raster graphics.
When zooming in or enlarging a bitmap image, the pixels are stretched and made into larger blocks. This is why bitmap images appear as poor quality when enlarged too much.

Vectors
A vector image uses scalable shapes such as straight lines and curves, using coordinates and geometry to precisely define the parts of the image. It is more efficient than bitmaps at storing large areas of the same colour because it does not need to store every pixel as a bitmap does.

Vector graphics can be scaled without losing resolution. They can be enlarged or reduced in size - but the file size will stay almost exactly the same.
One of the most common vector file formats is scalable vector graphics (SVG). SVG is an open standard for vector graphics.

Audio


Sound itself is a continuously varying (analogue) data. However if we want to store it in a computer it must be converted to binary (digital) data.
The analogue sound data is sampled at set intervals and the values that are sampled are used to represent the sound in digital format.


The sample rate determines the quality of the sound rerecorded. This is how often we take a sample in a given time. The more samples, the closer the digital sound is to the original analogue sound, however more data will be required.

Another factor is the accuracy of the values. The bit rate is the number of bits per given time period available for each sample measured in kilobits per second.
