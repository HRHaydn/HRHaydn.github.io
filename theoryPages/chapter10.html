Computer Systems

The Central Processing Unit (CPU)

A CPU (or Central Processing Unit) carries out instructions given to it. A CPU is comprised of transistors that can be combined to build logic gates, which in turn build the circuitry inside a processor.

The CPU carries out instructions in a series of Fetch, Decode and Execute cycles, where data is retrieved, decoded for instructions and then executed by the CPU.

With increasing speeds, smartphones today can have faster processors than those of desktop computers just ten years ago. This is down to Moore's law.

Clock Speed
Processors work at incredibly high speeds, performing cycles of Fetch, Decode and Execute over a billion times in less than a second.

Moore's Law


Gordon Moore, founder of Intel®, predicted in 1965 that the number of transistors would double approximately every two years. This trend is described as Moore’s Law. Doubling the number os transistors (into the same space) increases the speed of the processor.

Although this trend is beginning to show an end as physical limitations begin to become more apparent, there is still evidence of the effects of Moore’s Law as processor speed continues to grow exponentially.

Inside The CPU


What is a register?
A register is a memory location within the CPU.
The registers themselves are extremely fast and can be used without causing a bottleneck (the slowest part of the system that limits the speed as a whole).



Program Counter:
Keeps track of the memory location of the line of machine code being executed. It gets incremented to point to the next instruction, with each cycle of the fetch-decode-execute cycle allowing the program to be executed in sequence one by one.
The program counter is also affected by instructions which alter the flow of control - for example LMC has BRZ, BRP and BRA.

Memory Data Register (MDR):
Stores the data that has been fetched from or stored in memory.

Memory Address Register (MAR):
Stores the address of the data or instructions that are to be fetched from or sent to.

Current Instruction Register (CIR):
Stores the most recently fetched instructions, waiting to be decoded and executed.

Accumulator (ACC):
Stores the results of calculations made by the ALU.

General Purpose Registers:
Processors can also have general purpose registers, which can temporarily store data rather than sending data to and from the comparatively much slower memory.



Buses:
Buses are communications channels through which data can be sent around the computer. There are three buses inside the computer we need to consider:
Data Bus - carries data between the processor and memory.
Address Bus - carries the address of the memory location being read from or written to.
Control Bus - sends control signals from the control unit.


Arithmetic Logic Unit (ALU):
Carries out calculations and logical decisions. Results are stored in the accumulator.

Control Unit (CU):
Sends out signals to coordinate how the processor works. It controls the how data moves around parts of the CPU and how it moves between the CPU and memory. Instructions are decoded in the control unit.

Improving CPU performance

Cache Memory
RAM is faster when compared to the speed of secondary storage, however it is still slower in comparison the CPU - making RAM a bottleneck in the speed at which processor can operate.
To improve upon this, a small amount of fast memory is built into the processor itself reducing the distance data has to travel - this is Cache Memory. By anticipating regularly used data and moving them to Cache, the overall speed of the processor can be improved. As well as being expensive however, larger sizes of cache work slower and therefore often modern computers have multiple levels of cache.

Multiple Cores:
A CPU can have multiple cores, the most common being “dual core” and “quad core” processors. Each core is a distinct processing unit on the CPU, as well as having its own cache the cores also share a higher-level cache that works directly with RAM.
When multitasking, each core can work on different applications. Through parallel processing it is also possible for multiple cores to work on the same task. Four cores does not mean the processor will work four times as fast.

Pipelining:
Pipelining is a method used within processors, with one instruction being executed, the next instruction being decoded all while the the one after being fetched from memory. Sometimes it is not able to be accurate when predicting what instruction needs to be fetched and decoded next. For example we might need to branch to another instruction. This is called flushing the pipeline. The more we have to the flush the pipeline, the less benefit pipelining gives us.

Imagine a manual car wash: one person fetches the water, another cleans the car and finally someone rinses the car off.

Fetch Water Clean Car Rinse Off
Step One Car One
Step Two Car Two Car One
Step Three Car Three Car Two Car One
Step Four Car Four Car Three Car Two

Graphics Processing Unit (GPU):
A GPU is a processing unit specifically designed to perform calculations associated with graphics. They have instruction sets specifically designed for the sorts of calculations required in graphics processing.
Often when rendering graphics, the same process needs to be performed for multiple points on the screen. To speed this up, a GPU has the ability to process these data in parallel; what is referred to as single instruction multiple data (SIMD).

Uses of GPUs include: modelling physical systems, audio processing, breaking passwords and machine learning.

Input, Output, Storage and Memory


Input Devices allow data to be entered into a computer. Examples include keyboards, mice, microphones, scanners and joysticks.
Output Devices allow information to be retrieved from a computer. Examples include printers, speakers, monitors and actuators (devices that cause movement).
