Problem Solving

How to solve a problem


So many problems...
Computing has a long history alongside with solving problems. However it is important to note how to tackle a problem, especially when devising solutions.

One manner in which you could go about solving a problem was listed by George PÃ³lya.

Understanding the problem

  • What do we know about the problem?
  • What information do we have?
  • Can the problem be divided into sub-problems? (This is known as 'problem decomposition')
  • Can we represent the problem abstractly, with a diagram or variables?

Devise a plan

  • Break it down, produce sub-problems.
  • Look for patterns.
  • Be creative and use intuition, but keep in mind the problem that needs to be solved.
  • Try solving a similar problem if the current one proves too difficult.


Carry out your plan
Make sure to evaluate the solution as you go. If the plan isn't working then starting again is not a bad option. Don't dig if you're in a hole.


Look back over what you have done
Often when a solution is made, it is easy to say the work is done and to move on. It is important to evaluate what we have accomplished and decide what could have been done better. This is necessary especially with algorithms in computing.
