Search and Sort Algorithms



Search Algorithms
There are two main types of search algorithms: Linear search and Binary search.

Linear search involves moving from item to item in a list and checking if the item is the desired one. If its not then carry on moving until you reach the end. The search is the simplest you can get. The benefit of this longer search is that it can work with an unsorted list.

Binary search works by dividing the list in two and seeing if the wanted value is in the upper half or lower half. This continues until the wanted value is found. This process is a lot faster than the linear search with larger lists, however the list needs to be sorted.

Sorting Algorithms
As demonstrated by the binary search, in order to do various other operations with data, the set must first be sorted. The ones that will be covered are bubble sort, insertion sort, merge sort and quick sort.


The first sorting technique is the bubble sort. This sorting method works by moving through the set of data, and compares the current element to the next element, switching positions if the first is larger. This in turn moves the largest element to the end, then on the next cycle the second largest, continuing onwards until the set is sorted.
This method is the easiest sorting method to understand and implement, however it is very inefficient compared to alternatives.


Another method of sorting is the insertion sort. This sort works by taking an item from an unsorted part of the list, comparing it to two items already sorted and inserting it into its correct positioning. This method is then repeated until the list is sorted.


Alternatively there is merge sort. Merge sort involves a technique revolving around divide and conquer. Merge sort first begins by breaking down a list into a set of lists containing one element. It then builds up these lists into one single list in an ordered format.


The final sorting algorithm that will be discussed is Quicksort. This algorithm works by using described “pivots”. A list is taken and the starting element put into a single list containing itself. Split the remainder of the list into two sublists, those greater and those less than or equal to the pivot. From this recursively split the sublists until they are all pivots. Combine the remaining pivots to generate a sorted list.

Complexity


When we talk about differences between algorithms, we can evaluate their efficiency by how long they take to execute and how much memory they use. However in the ever evolving computing industry, computers are roughly doubling in processing power every 18 months. With this in mid there needs to be a more useful way to compare algorithms in their complexity.

This is achieved with the Big-O notation. It is written as O(x) where x is the worst case scenario. We are not bothered about values with the complexity, as it is the scaling that we are concerned about.
There are 5 kinds of complexity that will be covered.

Constant complexity O(1)
This is where the complexity is constant. No matter the size of the data set, the time taken or memory used is constant. Take for example removing the first item from a list, no matter the size of the list, it will take the same time.

Linear complexity O(n)
Algorithms with linear complexity increase at the same rate as the input size increases. An example of this is the average time taken to find an item using linear search, hence the name.

Exponential complexity O(n^k) (where k>1)
These do not scale well at all. This means that as the data size increases, the rate at which the time increases gets larger. An example of this type of complexity would be the bubble sort.

Logarithmic complexity O(logn)
Logarithms put simply are the opposite of exponentials, meaning the rate by which they increase is reduced as the data size gets larger. Therefore they scale the best with an increasing data size. An example of this type of complexity would be the binary search.
