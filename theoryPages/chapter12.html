Questions:

  1. 10100110 + 110011 = ?
  2. 1111 + 1001 = ?
  3. 10010 - 1011 = ?
  4. 10000 - 1111 = ?
  5. 10101010 - 10111 = ?


Answers:

  1. 10100110 + 110011 = 11011001
  2. 1111 + 1001 = 11000
  3. 10010 - 1011 = 0111
  4. 10000 - 1111 = 00001
  5. 10101010 - 10111 = 10010011


Adding using two's complement
Adding in two complement is similar to that of normal binary numbers. However when adding large binary numbers we can encounter an interesting issue. If two large numbers are added with a value that exceeds the 8-bits then the overflow will result in a negative number.
Take for example the addition of 103 + 115. In binary this would be 01100111 + 01110011 which would result in 11011010. However this binary number is represented in two's complement and as such returns a value of -38.
During addition, if the calculation were to result in a number that was too small to represent, then this would be called underflow.

Subtracting using two's complement
Subtracting again is very similar again to ordinary binary subtracting, but with a twist.
Take for example 7-1 in binary with binary values of 0111 and 0001 respectively.
Firstly we need to convert the 1 to its two's complement version.
This can be done with the simple process of using one's complement. This involves changing all the 1's to 0's and all the 0's to 1's then adding 1.
When we do this to 0001 we get 1111 which is equal to -8+4+2+1 = -1.

Now we have 0111 and 1111 we add them together.
This makes the binary number 10110. This results in an overflow error. When working in two's complement, if we encounter an overflow, we discard the carried bit.
The final binary number is therefore 0110 which equates to 6.
